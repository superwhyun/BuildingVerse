- 코드를 수정하기 이전에 어떤 수정이 필요한지 먼저 분석한 이후에 수정 방향을 사용자에게 제공하고, 사용자의 허락이 있는 경우에만 코드 수정을 진행한다.
- 코드 수정이 필요한 경우, 어떤 파일을 수정할 것인지 먼저 나열하여 사용자의 확인을 받도록 하며, 허락되지 않은 파일에 대해서는 수정하지 않는다.
- 사용자가 특정 기능 개발을 요구할 경우 바로 코드를 수정하지 말고, 그 기능 개발을 위한 계획을 먼저 세우고, 사용자에게 계획에 대한 허락을 받고 코드 개발 진행할 것.
- 반드시 코드수정 들어갈때는 명시적으로 '진행할까요?'로 물어보고 해. 절대 사용자 허락없이 고치지 말것.
